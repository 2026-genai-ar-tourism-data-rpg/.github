# 도깨비: 팔도의 비밀 — 사용자 시나리오 기획서

> 이 문서는 기술 구조나 "AI를 어디에 쓰는가"가 아니라, **실제 사용자가 앱을 켜서 무엇을 하고 어떻게 느끼는가**를 시간순으로 기술한다.
> 무대는 제안서의 파일럿 지역인 **서울 종로**, 예시 사건은 **「북촌의 사라진 도깨비불」**.

---

## 0. 한 줄로

> **"포켓몬 GO처럼 밖에 나가서 돌아다니는데, 메이플스토리처럼 퀘스트를 받고 NPC랑 대화하고 친구랑 파티 짜서 사건을 푸는 게임."**

핵심은 두 가지다.
1. **발로 움직여야 게임이 진행된다.** (실제 관광지 = 게임 맵)
2. **혼자도 재밌고, 친구랑 하면 더 재밌다.** (메이플의 솔로 사냥 ↔ 파티 퀘스트 구조)

---

## 1. 핵심 게임 루프 (한 사건을 푸는 사이클)

```
지도에서 단서 위치 확인
      ↓
실제로 그 장소까지 이동 (GPS 반경 진입)
      ↓
도깨비 NPC 등장 → 대화로 사건 정황 파악
      ↓
AR 카메라로 주변 스캔 → 숨은 단서 오브젝트 발견·수집
      ↓
단서 조합 → 다음 장소 해금 (또는 추리 입력)
      ↓
사건 해결 → 보상(도깨비불·도감·칭호) → 다음 사건/지역 해금
```

이 루프 한 바퀴가 메이플의 "퀘스트 수락 → 사냥/수집 → NPC에게 보고 → 보상"과 같다.

---

## 2. 단일 유저 시나리오 (솔로 플레이)

> 메이플 비유: **혼자 사냥터 돌면서 퀘스트 깨는 흐름.** 내 페이스대로, 내 동선대로.

### 등장인물: 지민 (24세, 주말에 종로 나들이)

**① 앱 실행 — "근처에 뭐가 있지?"**
- 지하철 3호선 안국역에서 내려 앱을 켠다.
- 지도에 현재 위치가 잡히고, 주변 관광지 위에 **도깨비불 마커**가 떠 있다.
- 마커는 두 종류로 보인다.
  - 🔵 **일반 단서** (경복궁·인사동 같은 유명지)
  - 🟣 **희귀 단서** (운현궁·서울공예박물관 같은 **사람 적은 곳** → 보상 가중치 높음)
- 화면 상단에 메인 사건 배너: **「북촌의 사라진 도깨비불 — 9개의 불씨 중 1개 발견됨」**

**② 첫 장소 도착 — 퀘스트 자동 활성화**
- 지민이 운현궁 반경(개방 공간 100m)에 들어선다.
- 진동 + 알림: *"도깨비의 기척이 느껴진다…"*
- 화면에 **도깨비 NPC**가 등장하고 말을 건다.
  > "여기가 흥선대원군이 살던 곳이라네. 헌데 어젯밤 도깨비불 하나가 이 담장 어딘가로 숨어버렸지. 자네, 눈썰미가 좋은가?"
- NPC 대사는 **그 장소의 실제 역사·설명 데이터를 바탕으로** 생성되므로, 대화만 따라가도 운현궁이 어떤 곳인지 자연스럽게 알게 된다.

**③ AR 탐색 — "어디 숨었지?"**
- "주변 둘러보기" 버튼 → 카메라 실행.
- 휴대폰을 들고 실제 담장·마당을 비추면, 화면 속 현실 위에 **반짝이는 단서 오브젝트(도깨비불, 발자국, 깨진 기와 등)**가 떠 있다.
- 지민이 몸을 돌려 오브젝트를 화면 중앙에 맞추고 탭 → **단서 획득**.
- 단서: *"도깨비불은 '더 오래된 곳'을 좋아한다."* → 다음 장소 힌트.

**④ 추리 & 이동**
- 획득한 단서로 NPC와 한 번 더 대화하거나, 단서를 조합한다.
- "더 오래된 곳" → 지도에서 창덕궁 마커가 반짝이며 해금된다.
- 지민은 실제로 창덕궁 쪽으로 걸어간다. (← 이 물리적 이동 자체가 게임 진행 = 관광 분산 유도)

**⑤ 사건 해결 & 보상**
- 몇 개 장소를 돌며 단서를 모으면 마지막에 **추리 정리 화면**.
- 정답을 맞히면 **도깨비불 1개 회수**, 사건 클리어.
- 보상:
  - 방문한 관광지가 **도감/스탬프**에 영구 기록
  - 사람 적은 곳(운현궁)에서 얻은 단서는 **희귀 보상 + 추가 경험치**
  - 지역 칭호 (예: "종로 초행 탐정")
- "다음 사건은 인사동에서…" → 자연스러운 다음 동선 제시.

> **솔로 플레이의 핵심 감정**: *내 발로 직접 찾아낸 장소*, *NPC가 들려준 그곳의 이야기*가 기억에 남는다. 단순 "관광지 리스트 보기"와 결정적으로 다른 지점.

## 3. 친구와 함께 시나리오 (멀티 / 파티플레이)

> 메이플 비유: **파티 퀘스트(파퀘).** 혼자선 못 깨는 걸 역할 나눠서 같이 깬다. + 길드 채팅의 왁자지껄함.

### 등장인물: 지민 + 친구 3명 (총 4인 파티 — 제안서상 최대 인원)

### 3-A. 협력 모드 (같이 사건 풀기)

**① 파티 구성**
- 지민이 파티를 만들고 **초대 코드/QR**를 친구에게 공유.
- 친구들이 입장 → 같은 사건 「북촌의 사라진 도깨비불」에 함께 투입.
- 메이플 파티창처럼 **상단에 4명의 위치·상태·획득 단서 수**가 실시간으로 보인다.

**② 역할 분담 — "흩어져서 찾자"**
- 한 사건의 단서가 **여러 장소에 분산**되어 있다.
  - 지민 → 운현궁
  - 친구A → 창덕궁
  - 친구B → 북촌한옥마을
  - 친구C → 서울공예박물관 (← 일부러 비인기지에 핵심 단서 배치)
- 각자 다른 장소로 흩어져 동시에 AR 탐색.
- **단서는 실시간으로 파티 전체에 공유**된다. (내가 찾은 단서가 친구 화면에도 즉시 뜸)

**③ 협력이 필요한 순간 — "이거 너희 단서랑 합쳐야 풀려"**
- 마지막 관문은 **4명의 단서를 모두 모아야** 풀리는 합동 추리.
- 예: 운현궁 단서(시간) + 공예박물관 단서(문양) + 한옥마을 단서(방향) + 창덕궁 단서(이름)를 조합해야 도깨비불의 정체가 드러남.
- 인게임 채팅으로 *"내 단서 '신시(申時)'인데 너네 뭐 나옴?"* 하며 맞춰본다.
- 다 모이면 파티 전원 동시 클리어 → **공동 보상 + 파티 보너스**.

**④ 같이 있을 때의 재미 포인트**
- 실시간 채팅 / 위치 공유로 "야 너 어디야" → 실제로 만나서 같이 다니기도.
- 한 명이 못 찾고 헤매면 다른 사람이 힌트 단서를 **건네줄 수 있음** (메이플 아이템 거래 느낌).

### 3-B. 경쟁 모드 (누가 먼저 푸나)

> 메이플 비유: **같은 사냥터에서 경쟁, 랭킹 보드.**

- 같은 사건을 **개별로 동시 시작**.
- 먼저 단서 다 모아 사건 해결한 사람이 1등.
- 실시간 **랭킹 리더보드** (현재 누가 몇 개 단서 모았는지 순위 표시).
- 같은 장소에 동시에 도착하면 "선점" 같은 가벼운 견제 요소.
- 보상: 순위별 차등 (1등 희귀 칭호 등).

## 4. 메이플스토리 대응표 (감 잡기용)

| 메이플스토리 | 도깨비: 팔도의 비밀 |
|---|---|
| 사냥터 / 맵 | 실제 관광지 (종로 일대) |
| 캐릭터 이동 | **내 실제 이동 (GPS)** |
| NPC 퀘스트 수락 | 도깨비 NPC 대화로 사건 받기 |
| 몬스터 잡고 아이템 드랍 | AR 카메라로 단서 오브젝트 수집 |
| 파티 퀘스트 (파퀘) | 4인 협력 모드 (단서 분담→합체 추리) |
| 파티창 / 길드 채팅 | 실시간 파티 위치·단서 공유 + 인게임 채팅 |
| 보스 레이드 | 한 지역의 메인 사건(여러 장소 합동 클리어) |
| 레어 드랍 / 보스 명성 | **저방문 관광지의 희귀 보상 가중치** |
| 도감 / 칭호 | 방문 관광지 스탬프 도감, 지역 칭호 |
| 시즌 이벤트 | 행사·축제 기간 한정 시즌 퀘스트 |

## 5. 사용자 여정 (온보딩 → 리텐션)

| 단계 | 솔로 | 친구와 |
|---|---|---|
| **첫 진입** | 튜토리얼 사건 1개 (가까운 1개 장소, 5분) | — |
| **첫 클리어** | 도깨비불 1개 + 첫 칭호 | 파티 초대로 유입 |
| **반복 동기** | 미수집 도감, 다음 지역 해금 | "같이 깨자" 약속, 랭킹 경쟁 |
| **재방문 유도** | 시즌/축제 한정 사건, 비인기지 희귀 보상 | 파티 보너스, 길드성 모임 |

## 6. 이 시나리오가 제안서 목표와 맞물리는 지점

- **관광 분산** → 핵심 단서를 일부러 비인기지(운현궁·공예박물관 등)에 배치, 저방문지 희귀 보상. 유저는 보상 때문에, 결과적으로 숨은 명소로 이동.
- **참여형 경험** → 보고 끝나는 게 아니라 *찾고·대화하고·추리하고·협력*한다.
- **데이터의 콘텐츠화** → 관광지 역사·설명이 NPC 대사·단서·사건의 재료가 되어, 게임을 깨는 동안 그 장소를 자연스럽게 알게 된다.

## 7. 맞춤형 시나리오 + 공용화 설계 (위시리스트 기반)

> 사용자가 "꼭 가고 싶은 곳"을 미리 입력하면, 그걸 바탕으로 시나리오를 짜주되 **숨은 관광지 방문 유도라는 목표가 깨지지 않게** 한다. 그리고 그렇게 만든 시나리오를 **다른 사람도 즐길 수 있는 공용 콘텐츠로** 승격시킨다.

### 7-1. AI를 "백지 창작"이 아니라 "검증된 블록 조립"으로

> 이것이 *"AI를 얼마나 쓰고, 시나리오를 얼마나 저장할지"* 에 대한 답이다.

관광지 하나하나를 **노드(블록)** 로 미리 만들어 DB에 저장한다.

```
[관광지 노드]
  ├─ 좌표 (TourAPI mapX, mapY)
  ├─ 카테고리 코드 → 퀘스트 유형
  ├─ 설명·역사 텍스트 (NPC 대사 컨텍스트)
  ├─ NPC 성격 시드 (말투/캐릭터)
  ├─ 단서 슬롯 (이 장소에 배치 가능한 단서 종류)
  └─ 혼잡도/방문도 (인기 ↔ 비인기 라벨)
```

| 역할 | 누가 | 비고 |
|---|---|---|
| **저장(DB)** | 관광지 노드/블록, 단서 템플릿, NPC 시드 | 검증된 재료. 품질·정확성 보장 |
| **생성(AI)** | 노드 **선택 + 배열** + 연결 스토리/대사 | 백지 창작 ❌ → 재료 조립 ⭕ |

이렇게 하면:
- **품질 통제** — AI가 엉뚱한 역사를 지어내지 않음(검증된 노드만 사용)
- **분산 통제** — 노드 선택 단계에서 비인기지 비율을 규칙으로 강제
- **재사용** — AI가 만든 "노드 시퀀스 + 대사"를 그대로 저장하면 = 완성된 시나리오 → 공용화 가능

### 7-2. 위시리스트를 분산 목표와 충돌 없이 끼우기 — "앵커 + 샛길" 모델

| | 역할 | 보상 |
|---|---|---|
| **앵커** (사용자 위시리스트, 유명지) | 시나리오의 **피날레 / 최종 목적지** | 일반 |
| **샛길** (시스템이 끼운 비인기지) | 앵커를 **해금하기 위한 단서 장소** | **희귀 (가중치 ↑)** |

**핵심: 가고 싶은 곳은 보내주되, 가는 길에 숨은 곳을 거치게 만든다.**

**예시 흐름**
- 사용자가 위시리스트에 입력: **경복궁, 광장시장** (둘 다 유명지)
- AI가 생성: 「**두 곳을 잇는 종로 탐정 코스**」
  - 시작: 운현궁 (비인기 · 단서 1)
  - 경유: 서울공예박물관 (비인기 · 단서 2)
  - 경유: 익선동 골목 (비인기 · 단서 3)
  - **피날레: 경복궁** (위시리스트 · 마지막 추리·보상)
  - 보너스 사건: 광장시장 (위시리스트 · 식도락 연계 보상)
- 사용자는 *가고 싶던 경복궁·광장시장을 결국 가지만*, 그 사이 비인기지 3곳을 자연스럽게 거친다.

### 7-3. 시나리오 "생성 규칙(계약)" — 분산 목표를 지키는 가드레일

AI가 시나리오를 생성할 때 반드시 지켜야 하는 제약. 어기면 거부/재생성.

- ✅ 비인기지(저방문) 노드 **최소 N개** 포함 (예: 단서 장소의 절반 이상)
- ✅ 위시리스트(유명지) 비중 **상한** (예: 전체 장소의 1/3 이하)
- ✅ 비인기지 단서 = **희귀 보상 가중치 유지**
- ✅ **도보 동선 현실성** (장소 간 거리·이동 시간이 현실적인가)
- ❌ 유명지만 줄줄이 연결한 코스 → 시스템이 자동 거부/보정
- ✅ 위시리스트가 비어 있으면 → 100% 시스템 큐레이션(=공식 시나리오와 동일)

### 7-4. 공용 시나리오 라이프사이클 (개인 → 모두의 콘텐츠)

```
[개인 맞춤]            [검수]                 [공개]
나만의 시나리오  ──▶  AI 자동 검증 + 큐레이션  ──▶  공용 시나리오 풀
(위시리스트 기반)      · 생성 규칙(7-3) 충족?         · 누구나 플레이
                      · 동선 실현 가능?              · 평점 / 즐겨찾기
                      · 부적절 텍스트 필터           · 리믹스(fork) 가능
```

- **개인** — 위시리스트로 만든 시나리오를 나 혼자(또는 파티) 플레이
- **검수** — 7-3 규칙 자동 검증 통과 + (필요 시) 운영 큐레이션
- **공개** — 공용 풀에 등록. 다른 사용자가 검색·플레이·평점, 마음에 들면 **자기 버전으로 리믹스**
- 이는 제안서 4)의 **UGC(사용자 생성 퀘스트) 확장**과 직결

### 7-5. 시나리오 3종 정리

| 종류 | 누가 만드나 | 분산 통제 | 용도 |
|---|---|---|---|
| **공식 시나리오** | 우리(운영) | 100% 큐레이션 | 파일럿·시즌·축제 한정 |
| **맞춤 시나리오** | 사용자 위시리스트 + AI | 생성 규칙(7-3)으로 강제 | 개인/파티 플레이 |
| **공용 시나리오** | 맞춤 → 검수 승격 | 검수 통과분만 | 모두가 즐기는 UGC 풀 |

## 8. 세부 설계 — 입력 UX · 리믹스 · 품질관리

### 8-1. 위시리스트 입력 UX

> 원칙: **위시리스트는 필수가 아니라 선택(앵커).** 입력이 무거우면 안 된다. "막연한 사용자 → 구체적인 사용자" 스펙트럼을 모두 받는다.

**3단 입력 (막연 → 구체)**

| 방식 | 언제 쓰나 | 예 |
|---|---|---|
| 🔍 **검색** (메인) | 갈 곳을 정확히 안다 | "경복궁" 자동완성 선택 |
| 📍 **지도 핀** | 지역만 정해졌다 | "이 동네에서 놀래" 영역 지정 |
| 🏷️ **취향 태그** | 완전히 막연하다 | #고궁 #먹거리 #한적한곳 |
| ⏭️ **건너뛰기** | 다 맡긴다 | → 100% 시스템 큐레이션(공식과 동일) |

**더 중요한 입력 — 시나리오를 실제로 좌우하는 제약(컨텍스트)**

위시리스트보다 동선·분량을 결정하는 건 이쪽이다. 가볍게 칩(chip) 선택으로 받는다.

- ⏱️ **가용 시간**: 2시간 / 반나절 / 하루
- 🚶 **이동수단**: 도보 / 대중교통 (→ 동선 반경 결정)
- 👥 **동행**: 혼자 / 친구와 (→ 솔로/협력 분기)
- 🎚️ **난이도**: 가볍게 / 본격 추리

> 입력 화면 한 줄 카피: *"가고 싶은 곳 있어요? (없으면 도깨비가 알아서 안내할게요)"* — 위시리스트가 선택임을 분명히.

### 8-2. 리믹스(fork) 허용 범위 — 단계적 개방

> 원칙: **쉬운 리믹스를 기본으로 넓게 열고, 깊은 수정은 별도 모드로.** 단 어떤 리믹스든 공용 등록은 **7-3 생성 규칙 재검증 통과**가 조건.

| 레벨 | 무엇을 바꾸나 | 난이도 | 공용 등록 조건 |
|---|---|---|---|
| **L0 플레이** | (수정 없음) 그대로 즐김 | — | — |
| **L1 장소 교체** ⭐기본 | 앵커를 내 위시리스트로 swap, 구조 유지 | 가장 쉬움 | 규칙 자동 재검증 |
| **L2 동선 조정** | 장소 추가/제거·순서 변경·난이도 조정 | 보통 | 규칙 재검증 + 동선 검증 |
| **L3 작가 모드** | 스토리·NPC 대사까지 수정 | 본격 UGC | 규칙 + **텍스트 필터·사실성 검수** |

**공통 가드레일**
- 어떤 레벨이든 → 7-3 규칙(비인기지 비율·위시리스트 상한·도보 동선) 재검증
- **원작자 크레딧 유지**(attribution) + **리믹스 계보(fork tree)** 기록 → "이 시나리오는 OOO의 코스에서 갈라짐"
- L3는 부정확한 역사·부적절 내용 위험이 크므로 자동 텍스트 필터 + (필요 시) 운영 검수 필수

> 추천 출시 전략: **L1을 먼저 오픈**(안전하고 누구나 할 수 있음) → 반응 보고 L2/L3 순차 개방.

### 8-3. 공용 시나리오 품질관리 — 게이트 + 평점 (역할 분담)

> 자동검수와 커뮤니티 평점은 **경쟁이 아니라 2단 깔때기**다. 검수는 *통과/탈락*, 평점은 *통과한 것들의 순위*.

```
[1단계 등록 게이트 — 자동검수]        [2단계 노출 조정 — 커뮤니티]
공개 전 필수 통과                      공개 후 지속 반영
 · 7-3 생성 규칙 충족?                  · 평점 (별점)
 · 도보 동선 실현 가능?                 · 완주율  ← 동선 현실성의 진짜 지표
 · 부적절/부정확 텍스트 필터            · 즐겨찾기 / 리믹스 수
 · 작성자 본인 1회 완주 테스트          · 신고 누적 → 자동 비공개 + 재검수
        │                                      │
        ▼                                      ▼
   공용 풀 등록                          상위 노출 / 추천 정렬
```

**핵심 포인트**
- **완주율**을 핵심 지표로 — 별점은 취향이지만, "끝까지 깬 비율"은 동선·난이도가 현실적인지를 가장 정직하게 보여줌
- **콜드 스타트 보정** — 신규 시나리오는 평점이 없어 묻히므로 "🆕 신규" 탭/슬롯에 노출 기회 부여
- **신뢰 배지 3종** — 🤖 자동통과 / 🔥 인기(커뮤니티) / ✅ 공식인증(운영 큐레이션)
- **어뷰징 방지** — 자기 평점·완주 조작 방지, 신고 임계치 초과 시 자동 비공개
